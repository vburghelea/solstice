# Service Approach: Training and Onboarding

## Audience-Based Training Approach

### Training Audiences

Training is tailored to three primary user groups, each with distinct needs and responsibilities:

| Audience                | Role                                       | Training Focus                                                              |
| ----------------------- | ------------------------------------------ | --------------------------------------------------------------------------- |
| viaSport Administrators | Platform oversight, PSO support, analytics | Admin functions, cross-org reporting, user management, system configuration |
| PSO Reporters           | Data submission, compliance                | Form completion, file uploads, submission tracking, deadline management     |
| Data Stewards           | Data quality, imports                      | Bulk imports, validation, error resolution, data quality monitoring         |

### Training Delivery Model

Training combines self-service resources with live sessions:

| Method                     | Audience                    | Format                                    |
| -------------------------- | --------------------------- | ----------------------------------------- |
| In-app guided walkthroughs | All users                   | Interactive tutorials within the platform |
| Live training sessions     | viaSport admins, PSO admins | Video conference workshops                |
| Train-the-trainer          | viaSport staff              | Preparing viaSport to support PSOs        |
| Self-service help center   | All users                   | Searchable guides, FAQs, templates        |

### Training Phases

| Phase                   | Timing      | Participants       | Content                                                             |
| ----------------------- | ----------- | ------------------ | ------------------------------------------------------------------- |
| viaSport Admin Training | Weeks 13-14 | viaSport staff     | Full platform capabilities, admin functions, PSO support procedures |
| PSO Rollout Training    | Weeks 15-18 | PSO administrators | Core workflows, data submission, common tasks                       |
| Ongoing                 | Post-launch | All users          | New feature introductions, refresher sessions as needed             |

### viaSport Admin Training (Weeks 13-14)

viaSport administrators receive comprehensive training covering:

- Platform navigation and dashboard customization
- User management and role assignment
- Organization hierarchy management
- Analytics and reporting tools
- Import and export functions
- Audit log review
- Support request handling
- System configuration options

Training includes hands-on exercises in the sin-perf environment using realistic data.

### PSO Rollout Training (Weeks 15-18)

PSO training is delivered in cohorts to ensure adequate support:

| Week | Cohort Size | Support Level                             |
| ---- | ----------- | ----------------------------------------- |
| 15   | 15 PSOs     | High-touch, live sessions                 |
| 16   | 15 PSOs     | Live sessions with Q&A                    |
| 17   | 15 PSOs     | Live sessions with Q&A                    |
| 18   | 15 PSOs     | Live sessions, transition to self-service |

Each PSO training session covers:

- Account setup and login (including MFA)
- Dashboard orientation
- Form completion workflow
- File upload procedures
- Submission tracking
- Help resources and support process

## Help Desk and Ticketing Model

### Support Tiers

| Tier   | Channel                 | Scope                               | Response                                |
| ------ | ----------------------- | ----------------------------------- | --------------------------------------- |
| Tier 1 | In-app support requests | General questions, how-to inquiries | 24 hours                                |
| Tier 2 | Email                   | Technical issues, bug reports       | 24 hours (standard), 4 hours (critical) |
| Tier 3 | Direct escalation       | Critical issues, system outages     | 4 hours                                 |

### Support Hours

| Coverage        | Hours                                             |
| --------------- | ------------------------------------------------- |
| Standard        | Business hours, Pacific Time (Mon-Fri, 9am-5pm)   |
| Critical issues | Business hours, with monitoring for system alerts |

24/7 support coverage is available as an optional add-on.

### In-App Support System

Users can submit support requests directly from the platform:

| Field       | Purpose                                        |
| ----------- | ---------------------------------------------- |
| Category    | Question, issue, feature request, feedback     |
| Subject     | Brief description                              |
| Description | Detailed explanation with optional attachments |
| Priority    | User-indicated urgency                         |

### Ticket Workflow

```
User submits request
        ↓
Ticket created with unique ID
        ↓
Support team reviews and assigns
        ↓
Response provided (in-app and email notification)
        ↓
User can reply or mark resolved
        ↓
Ticket closed
```

### Response Commitments

| Priority | First Response | Resolution Target |
| -------- | -------------- | ----------------- |
| Critical | 4 hours        | Same business day |
| High     | 8 hours        | 2 business days   |
| Standard | 24 hours       | 5 business days   |
| Low      | 48 hours       | Best effort       |

### Escalation Path

| Escalation Trigger      | Action                                     |
| ----------------------- | ------------------------------------------ |
| Response SLA missed     | Automatic escalation to project lead       |
| Critical issue reported | Immediate notification to development team |
| viaSport escalation     | Direct access to Austin Wallace            |

### Knowledge Base Integration

Common support requests inform help center improvements:

- Frequently asked questions added to FAQ
- Recurring issues documented in troubleshooting guides
- Training gaps addressed in walkthrough updates

### Support Reporting

viaSport receives monthly support reports including:

- Ticket volume by category
- Response time performance against SLAs
- Common issue trends
- Resolution statistics
